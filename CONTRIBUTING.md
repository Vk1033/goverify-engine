# Contributing to GoVerify Engine

First off, thank you for considering contributing to GoVerify Engine! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue with:
- A clear title and description.
- Steps to reproduce the bug.
- Any relevant logs or screenshots.

### Suggesting Enhancements
We love new ideas! If you have a suggestion, please open an issue describing the feature and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the Go style guidelines (`go fmt`).
3. Add tests for any new functionality.
4. Issue a PR with a detailed description of your changes.

## Code Style
- Follow idiomatic Go patterns.
- Use `uber-go/fx` for dependency injection.
- Document public functions and types.
